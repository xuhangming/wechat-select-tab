# 登录授权优化说明

## 更新内容

已将抽签小程序优化为**必须授权登录后才能使用**的模式，采用微信小程序最新的授权方式。

## 主要变更

### 1. 首页登录引导 (pages/index/)

#### 未登录状态
- 显示登录引导页面
- 用户需要选择头像（点击头像区域，从相册选择或使用微信头像）
- 用户需要输入昵称（支持 type="nickname" 自动填充微信昵称）
- 点击"完成登录"按钮完成授权

#### 已登录状态
- 显示用户头像和昵称
- 显示"我创建的"和"我参与的"抽签列表
- 可以创建新抽签和查看详情

### 2. 数据加载逻辑

- **未登录时**：不加载任何抽签数据，只显示登录引导
- **已登录时**：自动加载用户创建和参与的抽签列表
- 用户信息保存在本地存储，下次打开自动登录

### 3. UI 设计

#### 登录引导页
- 居中显示的白色卡片
- 大号抽签图标 🎲
- 清晰的标题和说明文字
- 头像选择区域（圆形按钮，支持点击选择）
- 昵称输入框（圆角设计，支持自动填充）
- 醒目的渐变色"完成登录"按钮

#### 主界面
- 保持原有的现代化设计
- 渐变紫色背景
- 卡片式布局
- 标签切换功能

## 用户体验流程

1. 用户首次打开小程序 → 看到登录引导页
2. 点击头像区域 → 选择头像（可以选择微信头像或相册图片）
3. 点击昵称输入框 → 输入昵称（会自动提示微信昵称，可直接选择）
4. 点击"完成登录"按钮 → 验证通过后进入主界面
5. 可以正常使用所有功能（创建、参与、查看抽签）
6. 下次打开小程序 → 自动登录，直接进入主界面

## 技术实现

### 新版授权方式

采用微信小程序基础库 2.21.0+ 推荐的授权方式：
- 使用 `<button open-type="chooseAvatar">` 获取头像
- 使用 `<input type="nickname">` 获取昵称
- 替代已废弃的 `getUserProfile` API

### 文件修改

1. **pages/index/index.js**
   - 移除 `canIUseGetUserProfile` 检查
   - 添加 `onChooseAvatar()` 方法：处理头像选择
   - 添加 `onNicknameInput()` 方法：处理昵称输入
   - 添加 `completeLogin()` 方法：验证并完成登录
   - 修改 `onLoad()` 方法：只有已登录才加载数据
   - 修改 `onShow()` 方法：只有已登录才刷新数据

2. **pages/index/index.wxml**
   - 添加头像选择按钮（open-type="chooseAvatar"）
   - 添加昵称输入框（type="nickname"）
   - 添加"完成登录"按钮
   - 使用 `wx:if` 条件渲染登录引导或主界面

3. **pages/index/index.wxss**
   - 添加头像选择区域样式
   - 添加昵称输入框样式
   - 添加登录按钮样式
   - 保持主界面原有样式

## 注意事项

1. 用户信息存储在本地 Storage，清除缓存后需要重新授权
2. 微信版本需要支持新版授权 API（基础库 2.21.0+）
3. 头像选择支持从相册选择或使用微信头像
4. 昵称输入框会自动提示微信昵称，用户可以直接选择或自定义
5. 云函数仍然通过 openid 识别用户，无需额外修改
6. 分享链接功能不受影响，接收者仍需授权登录后才能参与

## 兼容性

- 基础库 2.21.0+：完整支持新版授权方式
- 旧版本：建议提示用户升级微信

## 优势

相比旧版 `getUserProfile` 方式：
1. 更符合微信官方推荐的最新规范
2. 用户体验更流畅，无需弹窗授权
3. 头像和昵称可以自定义，不强制使用微信信息
4. 避免使用已废弃的 API，保证长期可用性
